/SAMLIdentities

/SAMLIdentities/{principalType} get

get

get a list of SAML identities of the specified type (e.g. candidates")

get /SAMLIdentities/{principalType}

get a list of SAML identities of the specified type (e.g. candidates")

  • Request
  • Response

URI Parameters

  • principalType: required (string)

Query Parameters

  • order: (string)

    The order parameter specifies the fields that are used to sort the result set. The actual fields that can used for sorting varies by resource - for SAMLIdentities, it is id.

    Use commas to combine multiple sort fields. Use the minus sign (-) to make sorting descending rather than ascending (the default).

    Order is commonly combined with the filter and limit parameters to achieve paging.

    Ordering is undefined if you don't specify the order parameter.

    Example:

    return the first 20 widgets, ordered by id (ascending), with
id > 128
GET /widgets?order=id&filter=id%3E128&limit=20

return the first set of widgets, ordered by quantity on hand
(ascending), then name (descending), that are active
GET /widgets?order=quantityOnHand,-name&filter=active%3D%3Dtrue
  • filter: (string)

    The filter parameter applies an EMCAScript-like expression as per http://www.ecma-international.org/ecma-262/5.1/#sec-11 to filter the result set. The actual fields that can used for filtering, and the complexity of the expression that is supported varies by resource - for SAMLIdentities, it is:

    simple comparisons on the id field must be supported, e.g.:

    • id = 20
    • id > 10
    • id < 10

    .

    The filter expression must be URL encoded before being used in a URI.

    Example:

    return the first set of widgets, ordered by id (ascending), that
pass the filter:   active==true||(id>20&&status=='online') 
GET /widgets?order=id&filter=active%3D%3Dtrue%7C%7C(id%3E20%26%26status%3D%3D%27online%27)
  • limit: (integer)

    Limit specifies the maximum number of items that will be returned in this result set. Less items may be returned (for example, on the last page).

    For SAMLIdentities, limit must be less than or equal to 30. This is also the default.

  • fields: (string)

    Use the optional fields operator to specify a comma separated list of the exact set of fields you want the endpoint to return. Use the forward slash character to specify nested fields. For example:

    GET /widgets?fields=kind,items/id,items/name

Response:

{
  "kind": "widgets",
  "items": [
    {
      "id": 10223,
      "name": "Red widget"
    },
    {
      "id": "10224",
      "name": "Blue widget"
    }
  ]
}

    If you don't specify fields for SAMLIdentities, the default set of fields is:
    id, items/personDetail/email, items/personDetail/title, items/personDetail/firstName, items/personDetail/lastName

HTTP status code 200

Body

Type: application/json

Schema:

{
	"$schema":"http://json-schema.org/draft-04/schema#",
	"type":"object",
	"additionalProperties":false,
	"properties":{
		"kind":{
			"enum":[
				"samlIdentities"
			]
			
		},
		"items":{
			"type":"array",
			"items":{
				"oneOf":[
					{
						"$ref":"samlIdentity.json"
					}
				]
			}
			
		}
		
	},
	"required":[
		"kind",
		"items"
	]
}

/SAMLIdentities/{principalType}/{id} get

get

get a specific SAML identity by its principleType (e.g. candidates") and id within that

get /SAMLIdentities/{principalType}/{id}

get a specific SAML identity by its principleType (e.g. candidates") and id within that

  • Request
  • Response

URI Parameters

  • principalType: required (string)
  • id: required (string)

HTTP status code 200

Body

Type: application/json

Schema:

{
	"$schema":"http://json-schema.org/draft-04/schema#",
	"type":"object",
	"additionalProperties":false,
	"properties":{
		"id":"integer",
		"identity":{
			"type":"object",
			"additionalProperties":"false",
			"properties":{
				"principalDeveloper":{
					"type":"string",
					"description":"the developer who defines this principal type"
				},
				"principalType":{
					"type":"string",
					"description":"the type of principal, e.g. the developer TAS defines 'candidate' and 'employee'"
				},
				"principleSubTypes":{
					"type":"array",
					"items":{
						"type":"string",
						"minimum":1
					},
					"description":"further refinements of the principal type, e.g. the developer TAS defines subtypes of 'internal' and 'external' for the principal type of 'candidate'"
				},
				"entityId":{
					"type":"string",
					"description":"globally unique identifier of the identity provider (IdP) this principal authenticated via"
				},
				"nameId":{
					"type":"string",
					"description":"identifier (unique within IdP) of this principal"
				},
				"isActive":"boolean"
			},
			"required":[
				"principalDeveloper",
				"principalType",
				"entityId",
				"nameId"
			]
			
		},
		"personDetail":{
			"type":"object",
			"additionalProperties":"false",
			"properties":{
				"kind":{
					"enum":[
						"personDetail"
					]
					
				},
				"email":"string",
				"title":"string",
				"firstName":"string",
				"lastName":"string",
				"phone":"string",
				"lodcale":"string",
				"language":"string",
				"timeZone":"string"
			},
			"required":[
				"kind"
			]
			
		},
		"employeeDetail":{
			"type":"object",
			"additionalProperties":"false",
			"properties":{
				"kind":{
					"enum":[
						"employeeDetail"
					]
					
				},
				"companyRef":"string",
				"profileId":"integer",
				"roleI":"integer"
			},
			"required":[
				"kind"
			]
			
		}
		
	},
	"required":[
		"identity",
		"personDetail"
	]
}

Example:

	"identity":{
		"principalDeveloper":"tas",
