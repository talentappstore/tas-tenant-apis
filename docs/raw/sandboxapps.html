About the sandbox apps

The sandbox apps allow you to develop your app without needing access to a full ATS.

Because the sandbox apps are private, you need to install them from an install token. This page lists the sandbox apps with their tokens. You don't normally need to install all of the apps below - each recipe will list the actual apps you need.

To install the sandbox apps you require, start by going to www.talentappstore.com, sign up and click at top right to create a tenant (use google auth to avoid a temporary issue with email-based logins).

Setup to create a job application

Follow the steps below. When installing apps, the apps that have links are unpublished sandbox apps that you install via their install token (see above). The other apps can be found publicly listed (find them with "Browse apps" from within your tenant). You can start in an incognito window for these steps.

  1. If you don't already have an account, sign in to www.talentappstore.com and create one (don't use email authentication for now).
  2. Create a tenant, then sign in to the tenant.
  3. (optional) If you want other people to be able to sign in to your tenant as users, then edit the regex on the installed identity app's "restrict sign ins from" section
  4. Install Developer ATS (when the 403 error occurs, click back, sign out and sign in again)
  5. Install the same identity app that is already installed in the tenant, but this time for principal type of candidate
  6. Install Developer Career Site
  7. Install Assessment hub
  8. Install Switchgear
  9. Install Tracker
  10. Install an install app, e.g. SimpleApply or Developer Apply
  11. Click to open Developer ATS
  12. Create a job (leave all the date fields blank, complete all other fields, make it visible externally)
  13. On the job details page, you'll see the embedded SwitchGear panel. Click "Edit" and drag the apply onto the job.
  14. Click to open Developer Careers
  15. Browse in to your job, click the Apply button, and complete the application process
  16. Return to Developer ATS and you should see the incoming job application (in the Incoming bucket)
