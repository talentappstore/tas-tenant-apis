Overview

This document is intended to help developers present their HR SaaS product as an onboarding or payroll (i.e. new hires) integration app in Talent App Store.

New hires integration can work on a push or pull model.

This document describes the push model, where the Applicant Tracking System (ATS) makes an API call against your server in real time every time there is a new hire.

Each new hire pushed to you includes a view - a detailed JSON document that can include fields from the job application, the job, the candidate, etc.

The customer controls the contents of the view using setup screens in the ATS, ensuring that each new hire includes all the data that you need for the integration.

See also:

  • The developer site, for defining your app.
  • The wiki, where you can get general information on building TAS apps.

Billing and financials

Billing for use of the onboarding API will be negotiated between the ATS vendor and onboarding vendor.

As for any app in Talent App Store, your app also pays for core API calls, as described at http://developer.talentappstore.com. These costs are low.

Your API endpoint

There are two possible endpoints your app might produce:

  • POST /applications/views/at/hire/now/byID/{application}/pushes
  • POST /applications/views/at/onboard/now/byID/{application}/pushes

Each incoming POST to your app's endpoint contains a view (a JSON document), with details about the new hire.

The view is customised by the customer to include only the fields that you need for the integration. They typically do this using a mapping UI in the ATS.

The mapping UI lets the customer (e.g. via drag and drop) add, remove and rename fixed and user-defined fields in the view, from any of:

  1. the candidate
  2. the job application
  3. hire details
  4. the job
  5. the hiring manager
  6. the recruiter

You can see from this example that a view can contain a lot of data. The actual data that will be included in the view that you receive is entirely controlled by the customer, using setup screens in the ATS.
