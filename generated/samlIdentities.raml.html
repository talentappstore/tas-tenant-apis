get /SAMLIdentities/{principalType}

get a list of SAML identities of the specified type (e.g. candidates")

  • Request
  • Response

URI Parameters

  • principalType: required (string)

Query Parameters

  • $orderby: (string)

    The $orderby query option is used to order the entries, where the resource is a collection. It behaves as per its namesake in the OData 2.0 Query String Options (http://www.odata.org/documentation/odata-version-2-0/uri-conventions/). The actual properties that can used varies by resource - for SAMLIdentities, it is id.

    Example:

    return widgets ordered by id (ascending)

GET /widgets?$orderby=id

return the first set of widgets, ordered by quantity on hand
(ascending), then name (descending)
GET /widgets?$orderby=quantityOnHand,name desc
  • $filter: (string)

    The $filter query option is used to identity a subset of the entries, where the resource is a collection. It behaves as per its namesake in the OData 2.0 Query String Options (http://www.odata.org/documentation/odata-version-2-0/uri-conventions/).

    The actual properties that can used for filtering, and the complexity of the expression that is supported varies by resource - for SAMLIdentities, it is:

    simple comparisons on the id field must be supported, e.g.:

    • id = 20

    • id > 10

    • id < 10

    .

    Example:

    return the widgets that are active, and that have status of 'online' or that are in Auckland:

GET /widgets?$filter=active eq true and (status eq 'online' or location/city eq 'Auckland')
  • $top: (integer)

    The $top query option is used to limit the number of entries, where the resource is a collection. It behaves as per its namesake in the OData 2.0 Query String Options (http://www.odata.org/documentation/odata-version-2-0/uri-conventions/).

    For SAMLIdentities, limit must be less than or equal to 30. This is also the default.

    Example:

    return the first 5 widgets:

GET /widgets?$top=5
  • $skip: (integer)

    The $skip query option is used to specify the start index of the entries, where the resource is a collection. It behaves as per its namesake in the OData 2.0 Query String Options (http://www.odata.org/documentation/odata-version-2-0/uri-conventions/).

    Example:

    return the widgets, starting with the third widget:

GET /widgets?$skip=2
  • $select: (string)

    The $select query option is used to specify which properties should be returned in the resource. It behaves as per its namesake in the OData 2.0 Query String Options (http://www.odata.org/documentation/odata-version-2-0/uri-conventions/). For example:

    GET /widgets?$select=kind,items/id,items/name

Response:

{
  "kind": "widgets",
  "items": [
    {
      "id": 10223,
      "name": "Red widget"
    },
    {
      "id": "10224",
      "name": "Blue widget"
    }
  ]
}

    If you don't specify $select for SAMLIdentities, the default set of fields is:
    id, items/personDetail/email, items/personDetail/title, items/personDetail/firstName, items/personDetail/lastName

    /SAMLIdentities/{principalType}/{id} get

    get

    get a specific SAML identity by its principleType (e.g. candidates") and id within that

    get /SAMLIdentities/{principalType}/{id}

    get a specific SAML identity by its principleType (e.g. candidates") and id within that

    • Request
    • Response

    URI Parameters

    • principalType: required (string)
    • id: required (string)

    HTTP status code 200

    Body

    Type: application/json

    Schema:

    {
	"$schema":"http://json-schema.org/draft-04/schema#",
	"type":"object",
	"additionalProperties":false,
	"properties":{
		"id":"integer",
		"identity":{
			"type":"object",
			"additionalProperties":"false",
			"properties":{
				"principalDeveloper":{
					"type":"string",
					"description":"the developer who defines this principal type"
				},
				"principalType":{
					"type":"string",
					"description":"the type of principal, e.g. the developer TAS defines 'candidate' and 'employee'"
				},
				"principleSubTypes":{
					"type":"array",
					"items":{
						"type":"string",
						"minimum":1
					},
					"description":"further refinements of the principal type, e.g. the developer TAS defines subtypes of 'internal' and 'external' for the principal type of 'candidate'"
				},
				"entityId":{
					"type":"string",
					"description":"globally unique identifier of the identity provider (IdP) this principal authenticated via"
				},
				"nameId":{
					"type":"string",
					"description":"identifier (unique within IdP) of this principal"
				},
				"isActive":"boolean"
			},
			"required":[
				"principalDeveloper",
				"principalType",
				"entityId",
				"nameId"
			]
			
		},
		"personDetail":{
			"type":"object",
			"additionalProperties":"false",
			"properties":{
				"kind":{
					"enum":[
						"personDetail"
					]
					
				},
				"email":"string",
				"title":"string",
				"firstName":"string",
				"lastName":"string",
				"phone":"string",
				"locale":"string",
				"language":"string",
				"timeZone":"string"
			},
			"required":[
				"kind"
			]
			
		},
		"employeeDetail":{
			"type":"object",
			"additionalProperties":"false",
			"properties":{
				"kind":{
					"enum":[
						"employeeDetail"
					]
					
				},
				"companyRef":"string",
				"profileId":"integer",
				"roleI":"integer"
			},
			"required":[
				"kind"
			]
			
		}
		
	},
	"required":[
		"identity",
		"personDetail"
	]
}

    Example:

    {
	"id":10267,
	"identity":{
		"principalDeveloper":"tas",
		"principalType":"employee",
		"nameId":"102ghfirhr784",
		"entityId":"acmeidp",
		"isActive":true
	},
	"personDetail":{
